Consultations

Utilisé pour permettre de modifier le contenu en rapport avec les consultations. Chaque requête possède une vérification des roles

Méthodes autorisées :     [GET, POST, PATCH, DELETE, OPTIONS]

GET  /consultations
Obtenir toutes les consultations

Pour un médecin ou un usager, supprime les consultation dont ils ne font pas partie

- Authentification

Nécessite le role de :  administrateur ,  secrétaire ,  mecedin ,  usager

- Requête

Méthode :    GET

URL :    /consultations

Header :

Authorization : Bearer eyJhbGc...
Réponse - 200

Header :

Content-Type : application/json

Body :

{
    "status": "success",
    "status_code": 200,
    "status_message": "[R200 REST API] : Consultations trouvées",
    "data": [
        {
            "id_consult": 1,
            "date_consult": "2024-01-01",
            "heure_consult": "08:00:00",
            "duree_consult": 15,
            "id_medecin": 1,
            "id_usager": 1
        },
        {
            "id_consult": 2,
            "date_consult": "2024-01-02",
            "heure_consult": "09:00:00",
            "duree_consult": 30,
            "id_medecin": 2,
            "id_usager": 2
        },
        ...
    ]
}

Réponse - 404

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 404,
    "status_message": "[R404 REST API] : Aucune consultation trouvée"
}

POST  /consultations
Créer une consultation

- Vérifications

  • Droit d'ajout
  • Données nécessaires : [id_medecin, id_usager, date_consult, heure_consult, duree_consult]
  • Médecin existe
  • Usager existe
  • Date
  • Consultation valide
  • Disponibilité usager/médecin

- Authentification

Nécessite le role de :  administrateur ,  secrétaire ,  mecedin ,  usager

- Requête

Méthode :    POST

URL :    /consultations

Header :

Content-Type : application/json
Authorization : Bearer eyJhbGc...

Body :

{
    "id_usager":"7",
    "id_medecin":"8",
    "date_consult":"12/10/2024",
    "heure_consult":"11:30",
    "duree_consult":"30"
}
Réponse - 201

Header :

Content-Type : application/json

Body :

{
    "status": "success",
    "status_code": 201,
    "status_message": "[R201 REST API] : Consultation ajoutée",
    "data": {
        "id_consult": 53,
        "date_consult": "2024-12-10",
        "heure_consult": "11:30:00",
        "duree_consult": 30,
        "id_medecin": 8,
        "id_usager": 7
    }
}

Réponse - 500

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 500,
    "status_message": "[R500 REST API] : Echec de l'ajout de la consultation"
}



PATCH  /consultations/{id}
Modifier une consultation

- Vérifications

  • Argument nécessaire
  • Type d'argument int
  • Droit de modification
  • Consultation existe
  • Données autorisées : [id_medecin, id_usager, date_consult, heure_consult, duree_consult]
  • Date (si présent)
  • Médecin existe (si présent)
  • Usager existe (si présent)
  • Consultation valide (si présent)
  • Disponibilité usager/médecin (si présent)

- Authentification

Nécessite le role de :  administrateur ,  secrétaire ,  mecedin ,  usager

- Requête

Méthode :    PATCH

URL :    /consultations/{id}

Header :

Content-Type : application/json
Authorization : Bearer eyJhbGc...

Body :

{
    "id_usager":"2",
    "id_medecin":"8",
    "date_consult":"12/10/2024",
    "heure_consult":"12:30",
    "duree_consult":"45"
}
Réponse - 201

Header :

Content-Type : application/json

Body :

{
    "status": "success",
    "status_code": 201,
    "status_message": "[R201 REST API] : Consultation modifiée",
    "data": {
        "id_consult": 6,
        "date_consult": "2024-12-10",
        "heure_consult": "12:30:00",
        "duree_consult": 45,
        "id_medecin": 8,
        "id_usager": 2
    }
}

Réponse - 500

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 500,
    "status_message": "[R500 REST API] : Echec de la modification de la consultation"
}
