Medecins

Utilisé pour permettre de modifier le contenu en rapport avec les médecins.

Allowed Methods :     [GET, POST, PATCH, DELETE, OPTIONS]

GET  /medecins/
Obtenir tout les médecins

- Authentication

Nécessite le role de :  administrateur ,  secrétaire ,  mecedin ,  usager

- Request

Method :    GET

URL :    /medecins/

Header :

Authorization : Bearer eyJhbGc...

- Response - 200

Header :

Content-Type : application/json

Body :

{
    "status": "success",
    "status_code": 200,
    "status_message": "[R200 REST API] : Médecins trouvés",
    "data": [
        {
            "id_medecin": 1,
            "civilite": "M.",
            "nom": "Dupont",
            "prenom": "Xavier"
        },
        {
            "id_medecin": 2,
            "civilite": "Mme.",
            "nom": "Darc",
            "prenom": "Jeanne"
        },
        ...
    ]
}

- Response - 404

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 404,
    "status_message": "[R404 REST API] : Aucun médecin trouvé"
}

POST  /medecins/
Créer un médecin

- Authentication

Nécessite le role de :  secrétaire , administrateur

- Request

Method :    POST

URL :    /medecins/

Header :

Content-Type : application/json
Authorization : Bearer eyJhbGc...

Body :

{
    "civilite":"M.",
    "nom":"Dupond",
    "prenom":"Gérard"
}

- Response - 201

Header :

Content-Type : application/json

Body :

{
    "status": "success",
    "status_code": 201,
    "status_message": "[R201 REST API] : Médecin inséré en base de donnée avec succès",
    "data": {
        "id_medecin": 11,
        "civilite": "M.",
        "nom": "Dupond",
        "prenom": "Gérard"
    }
}

- Response - 500

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 500,
    "status_message": "[R500 REST API] : Erreur lors de l'insertion du médecin en base de donnée"
}
