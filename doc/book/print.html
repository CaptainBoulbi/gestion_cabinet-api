API Application

Lien de l'api :

  • api_med_app.fruitpassion.fr

Pour n'importe quelle action l'api requiert un JWT. Une authentification sans compte peux se faire via le role   invite.

  • Voir comment se connecter en tant qu'invite

ℹ️ Note

Le symbole 🚹 signifie qu'une action n'est possible que sur son propre compte.

Par exemple un medecin ayant ✅ 🚹 à l'action Modifier, ne peux que modifier que des informations de son propre compte.

Medecins

Utilisé pour permettre de modifier le contenu en rapport avec les médecins.

Allowed Methods :     [GET, POST, PATCH, DELETE, OPTIONS]

GET  /medecins/
Obtenir tout les médecins

- Authentication

Nécessite le role de :  administrateur ,  secrétaire ,  mecedin ,  usager

- Request

Method :    GET

URL :    /medecins/

Header :

Authorization : Bearer eyJhbGc...

- Response - 200

Header :

Content-Type : application/json

Body :

{
    "status": "success",
    "status_code": 200,
    "status_message": "[R200 REST API] : Médecins trouvés",
    "data": [
        {
            "id_medecin": 1,
            "civilite": "M.",
            "nom": "Dupont",
            "prenom": "Xavier"
        },
        {
            "id_medecin": 2,
            "civilite": "Mme.",
            "nom": "Darc",
            "prenom": "Jeanne"
        },
        ...
    ]
}

- Response - 404

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 404,
    "status_message": "[R404 REST API] : Aucun médecin trouvé"
}

Statistiques

Vérifications

Répertoire de toute les vérifications pouvant être effectué pour chaque requete.


Méthode


Chaque route détient une liste de méthodes autorisée. Cette vérification est la première effectuée à chaque requête.

- Response - 405

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 405,
    "status_message": "[R405 REST AUTH] : Methodes utilisées non autorisées"
}

Argument


Utilisé quand la requete attend un argument que ca soit un {id} ou un nom de fonction tel que {usager} (pour les statistiques).

- Response - 400

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 400,
    "status_message": "[R400 REST API] : Aucun argument n\'a été fourni"
}

Type argument string


Utilisé pour vérifier le type d'argument que l'on attend est une string.

- Response - 400

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 400,
    "status_message": "[R400 REST] : L\'argument doit être une chaine de caractère"
}

Type argument int


Utilisé pour vérifier le type d'argument que l'on attend est un int (pour les {id}). Impose une taille limite de 2147483647.

- Response - 400

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 400,
    "status_message": "[R400 REST] : L\'identifiant doit être un entier positif non null"
}

- Response - 400

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 400,
    "status_message": "[R400 REST] : L\'identifiant doit être un entier inférieur à 2147483647"
}

Date


Utilisé pour vérifier si une date est inférieure (date de naissance) ou supèrieur (rendez-vous) à la date actuelle.

- Response - 400

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 400,
    "status_message": "[R400 REST API] : La date <date_parametre> est invalide car elle est inferieure à la date actuelle"
}

- Response - 400

Header :

Content-Type : application/json

Body :

{
    "status": "error",
    "status_code": 400,
    "status_message": "[R400 REST API] : La date <date_parametre> est invalide car elle est supérieure à la date actuelle"
}
